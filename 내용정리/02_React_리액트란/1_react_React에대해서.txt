리액트 공식문서에서는 UI(User Interface) 를 구현하기 위한 JavaScript 라이브러리 라고 되어있다.

JavaScript UI 라이브러리의 종류에는 대표적으로 3가지가 있는데 
AngularJS, React, VueJS 가 있다.

AngularJS, VueJS 는 프레임워크로 취급하고 React 는 라이브러리로 취급하는데 뭐가 다른 것일까?

    ★  프레임워크 vs. 라이브러리
        두 가지 모두 자주 사용하는 기능들을 모아놓은 것이다. 하지만 코딩 작성 흐름 제어의 권한이
        누구에게 있냐는 차이가 있다. 프레임워크는 개발자가 아닌 프로그램에 코딩 작성 흐름 제어의 권한이 있다.
        예를 들어 NestJS 가 있는데 NestJS 는 Module, Controller, Service 등을 제공하여 개발자에게 
        '이 흐름대로 코딩하세요~' 라고 알려주며 기초적인 틀을 제공해준다. 반대로 라이브러리는 개발자 마음이다.
        자꾸 백엔드 스택을 예로 들어서 좀 그렇지만... 
        정확히는 모르겠지만 express 가 그 예시인 것 같다. 어떤 흐름대로 코딩을 하든 개발자 마음이다.
